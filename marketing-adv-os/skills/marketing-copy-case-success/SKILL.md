# MARKETING COPY CASE SUCCESS

> Skill Tier 2 (Executor — Copy). Case study completo (1200-2500 palavras).

## OBJETIVO

Produzir case study estruturado e exportavel, com 6 secoes que respondem em sequencia: contexto + problema + solucao + resultado + voz do cliente + visualizacao. Formato adequado a sales enablement, propostas B2B, paginas /casos.

## PRE-REQUISITOS

Workspace + MEMORY. **Critico (mesmo gate de post-depoimento):**
- **Autorizacao escrita** do cliente (Track A: OBRIGATORIO; Track B: recomendado)
- **Dados quantitativos verificaveis** (Track A: temporais + qualitativos; Track B: numeros financeiros)
- **Fala literal** do cliente (testimonial)

## FLUXO

### 1. Gate de autorizacao + dados

```
GATE OBRIGATORIO:

1. Autorizacao escrita do cliente para uso publico do case?
   - SIM (especifique: e-mail, mensagem, documento assinado)
   - NAO -> BLOQUEIO Track A. Em Track B, e ainda assim recomendado.

2. Dados quantitativos verificaveis?
   - SIM (cite os numeros)
   - NAO -> case fica fraco. Sugerir adiar ate ter.

3. PII (CPF, processo, valor sensivel) que NAO pode aparecer?
   - Identificar e mascarar antes da producao.

4. Track A — VE-04: o case promete resultado generico?
   - Se case mostra "ganhei a acao", precisa disclaimer FORTE
   - Reformular: "obtivemos sucesso na tese X aplicada a [contexto]"
```

### 2. Coletar dados estruturados

```
DADOS DO CASE:

Cliente:
- Nome ou descricao funcional ("empresa de [setor], [faixa faturamento]" se anonimo)
- Segmento / nicho
- Tamanho ou contexto

Background (o cenario antes):
- Situacao da empresa/segurado(a) antes
- Por que procuraram voce / sua oferta
- Tentativas anteriores (se houve)

Challenge (o problema especifico):
- O nucleo do problema
- O que tornava dificil resolver
- Stakeholders envolvidos (em B2B)
- Constraints (tempo, orcamento, regulatorio)

Solution (o que voce/produto fez):
- Metodologia aplicada (sem expor estrategia processual sensivel em Track A)
- Etapas principais
- Time/produto envolvido
- Diferenciador da abordagem

Results (o resultado):
- Metrica primaria (numero verificavel)
- Metricas secundarias (se houverem)
- Tempo decorrido
- Resultados intangíveis (autonomia, clareza, paz)

Testimonial (fala do cliente):
- Citacao literal OU consolidada com aprovacao
- Pode ser bloco de 2-4 frases
- Nome + cargo (se autorizado)
```

### 3. Estrutura do case study

```
=== CASE STUDY: [Cliente / Tipo de cliente] ===

[HEADLINE]
"[Resultado/transformacao em ate 12 palavras]"
Ex: "Empresa de [setor] reduziu [metrica] em [X] em [tempo]"

[SUBHEADLINE]
"[Contexto especifico que reforca relevancia]"

[BLOCO 1 — BACKGROUND] (200-400 palavras)
"## Contexto

[Empresa/Cliente] [descricao + setor + porte]. [Situacao antes — sem julgamento, factual]. [Tentativas anteriores se houver]."

[BLOCO 2 — CHALLENGE] (200-400 palavras)
"## O desafio

[O nucleo do problema, especifico]. [Por que era dificil resolver]. [Stakeholders / constraints]. 'Citacao curta do cliente sobre o desafio'"

[BLOCO 3 — SOLUTION] (300-600 palavras)
"## A solucao

[Como voce/oferta entrou]. [Etapas principais, em ordem]. [Diferenciador da abordagem]. [Time/produto envolvido]."

Track A: SEM expor estrategia processual sensivel ou detalhes que quebrem sigilo de OUTROS clientes/casos similares.

[BLOCO 4 — RESULTS] (200-400 palavras)
"## Resultados

Em [tempo], [Cliente] alcancou:
- [Metrica primaria verificavel]
- [Metrica secundaria 1]
- [Metrica secundaria 2]
- [Resultado intangivel]"

[BLOCO 5 — TESTIMONIAL] (50-150 palavras)
"## Nas palavras de [Cliente]

'[Fala literal ou consolidada com aprovacao — 2-4 frases]'

— [Nome ou cargo + empresa, se autorizado]"

[BLOCO 6 — METRICS VISUAL] (sugestao de visualizacao)
"## Resumo visual

[Sugerir grafico/tabela/diagrama:
- Grafico antes/depois (linha temporal)
- Tabela comparativa (3 colunas: metrica / antes / depois)
- Diagrama do processo aplicado]"

[DISCLAIMER OBRIGATORIO Track A]
"_Resultados refletem o caso especifico de [Cliente] aplicado em [contexto]. Cada caso e unico. Este estudo nao constitui promessa de resultado em situacoes diferentes._"

[CTA]
- Para LP: "Quer entender se ha caminho parecido para o seu caso?" [Marcar conversa]
- Para pitch B2B: "Vamos discutir como aplicar isso ao seu contexto?" [Reuniao 30 min]
- Para blog: "Compartilhe esse case com quem precisa ver." [Compartilhar]
```

### 4. Versoes para canais

Gerar 3 versoes simultaneas:
- **LONGA (1500-2500 palavras):** completa para LP /casos e PDF download
- **MEDIA (700-1000 palavras):** para email broadcast ou newsletter
- **CURTA (200-400 palavras):** para sales enablement (1-pager) e pitch deck

### 5. Voz + autenticidade

Voz do MEMORY com TOM ANALITICO. Case study e formal mas humanizado. Inclui pelo menos 1 elemento de imperfeicao na secao Solution ("Tivemos que ajustar [X]" ou "Nao foi linear — em [momento] [Y]"). Imperfeicao = autenticidade = confianca.

### 6. Diretoria Criativa R2 + R3 + R4

R2: estrutura BG-CH-SO-RE-TE-MV clara + tom analitico + CTA singular.
R3:
- **VE-15 (Track A) — CRITICA:** autorizacao escrita? Senao, BLOQUEIO.
- **VE-04 (Track A) — disclaimer obrigatorio:** sem disclaimer, generalizacao = VIOLACAO.
- **VE-12:** numeros e citacoes verificaveis.
- **VE-14:** se case e composto/representativo, DECLARAR.
- **PII** mascarado mesmo com autorizacao.

R4: case study e sales enablement — KPI = leads qualificados gerados, conversoes em proposta comercial. NAO e curtida.

## OUTPUT

Case study em 3 versoes (longa/media/curta) + disclaimer + plano de uso por canal + veredito Diretoria + indicacao de visualizacao recomendada.

## VEDACOES ESPECIFICAS

- **VE-15 (Track A) — CRITICA:** Case real SEM autorizacao escrita = BLOQUEIO TOTAL
- **VE-04 (Track A):** Disclaimer obrigatorio + nao promete resultado generalizado
- **VE-12:** Numeros, datas, fala — verificaveis ou explicitamente compostos
- **VE-14:** Case composto -> declarar
- **PII:** CPF, processo, valor sensivel = MASCARAR sempre

## PROTOCOLOS ACIONADOS

- **2.1 Briefing** — gate de autorizacao + coleta estruturada de dados
- **2.3 Producao** — voz analitica + autenticidade (1 imperfeicao)
- **2.4 Compliance** — VE-15 e VE-04 (Track A) sao prioritarias
- **2.5 Mensuracao** — KPI = leads qualificados, nao engajamento
